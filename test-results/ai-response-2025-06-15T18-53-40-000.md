## Решение от ИИ

Проблема: Элемент не найден на странице. Возможные причины и решения:

1. **Элемент не загрузился вовремя**:
   - Увеличь таймаут ожидания: `await page.locator('button[data-testid="submit"]').waitFor({ timeout: 60000 });`
   - Добавь ожидание перед действием: `await expect(page.locator('button[data-testid="submit"]')).toBeVisible();`

2. **Неправильный селектор**:
   - Убедись, что `data-testid='submit'` корректен. Проверь в браузере через DevTools.
   - Попробуй альтернативный селектор, например, по тексту: `button:has-text("Submit")`.

3. **Элемент появляется после асинхронных операций**:
   - Добавь ожидание завершения загрузки данных, например:
     ```typescript
     await page.waitForResponse(response => response.url().includes('/api/load-data'));
     ```

4. **Элемент внутри iframe**:
   - Если кнопка внутри iframe, переключись на него:
     ```typescript
     const frame = page.frameLocator('iframe-selector');
     await frame.locator('button[data-testid="submit"]').click();
     ```

5. **Проверь, не блокирует ли элемент что-то**:
   - Убедись, что нет оверлеев или модальных окон, которые могут перекрывать кнопку.

Для e-commerce проектов на React также может быть полезно добавить ожидание завершения реактивных обновлений:
```typescript
await page.waitForFunction(() => document.fonts.ready);
```
